# Signed plugin fixtures

Fixtures generated by `scripts/sign-plugin-package.mjs --ephemeral`.
The ephemeral keypair is regenerated each time; both the `.sig.json` and
`ephemeral.pub.pem` must be committed together.

## Files

| File | Purpose |
|---|---|
| `bobsoft-linter-0.1.0.fpk` | Tiny fake package (clean — sha256 matches `sig.json`) |
| `bobsoft-linter-0.1.0.fpk.sig.json` | `ServedSignatureMetadata` signed by the ephemeral key |
| `ephemeral.pub.pem` | SPKI ed25519 public key for the ephemeral pair (committed; non-secret) |
| `bobsoft-linter-0.1.0-tampered.fpk` | Same package bytes but with the first byte flipped → sha256 mismatch |
| `bobsoft-linter-0.1.0-tampered.fpk.sig.json` | SAME `sig.json` as the clean package → manifest.contentSha256 ≠ sha256(tampered bytes) |

## Absent-signature case

Use `bobsoft-linter-0.1.0.fpk` with **no** `.sig.json` sidecar.
The test constructs that scenario by simply not providing a `ServedSignatureMetadata`.

## Regenerating

```sh
node scripts/sign-plugin-package.mjs \
  --in apps/desktop/src/main/firefly-plugin/install/__fixtures__/signed/bobsoft-linter-0.1.0.fpk \
  --key-id ephemeral-test-key \
  --namespace bobsoft --name linter --version 0.1.0 \
  --signed-at 2026-06-16T00:00:00.000Z \
  --ephemeral \
  --anchor-out apps/desktop/src/main/firefly-plugin/install/__fixtures__/signed/ephemeral.pub.pem
```
