- Task 17 hard blocker revisited in task 18: the prior finding "chromium 148 in LinuxServer/Selkies does not bind CDP" was a wrapper-level bug, not a binary limitation. The IP:PORT format `--remote-debugging-port=0.0.0.0:9222` is silently dropped by both chromium 148 and chrome 149; bare `--remote-debugging-port=9222` binds 127.0.0.1:[::1]:9222 inside the container. Combined with a socat relay init script (0.0.0.0:9222 -> [::1]:9222) so Docker Desktop's userland proxy on macOS can reach loopback binds, the same lscr.io/linuxserver/chromium:latest image now serves both stream and CDP from one chrome instance.
