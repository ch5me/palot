## 5. Queue Contract Mismatch (CRITICAL BUG)

### types.ts (pm-side-agents/types.ts:155-160):
```typescript
export interface Ch5PmSideAgentQueuePayload {
  jobs?: Ch5PmSideAgentQueueJob[];
  claims?: Ch5PmSideAgentQueueClaim[];
  updatedAt?: string | null;
  generatedAt?: string | null;
}
```
Assumes FLAT shape: payload.jobs, payload.claims

### ACTUAL live payload (curl proof):
```json
{
  "ok": true,
  "health": "degraded",
  "helper": "inbox",
  "rows": {
    "jobs": [...],
    "claims": [...]
  },
  "generatedAt": "...",
  "degradedReasons": [...],
  "dataAges": {...}
}
```
Real shape: ENVELOPE with payload.rows.jobs, payload.rows.claims

### composition.ts (pm-side-agents/composition.ts:251-259):
```typescript
const jobs = Array.isArray(payload.jobs) ? payload.jobs : []
const claims = Array.isArray(payload.claims) ? payload.claims : []
const partial = !Array.isArray(payload.jobs) || !Array.isArray(payload.claims)
```
WRONG: reads payload.jobs and payload.claims directly, but live payload has payload.rows.jobs and payload.rows.claims.

### Impact:
- composition.ts will always see jobs=[] and claims=[] (empty arrays)
- partial flag will always be true (triggers "queue-partial" degraded reason)
- Side Agents queue panel will show empty even when jobs/claims exist

### Fix required:
- Update Ch5PmSideAgentQueuePayload type to match live envelope shape
- Update composition.ts buildQueueSource to read payload.rows.jobs and payload.rows.claims
- Add regression test with fixture matching live envelope shape
