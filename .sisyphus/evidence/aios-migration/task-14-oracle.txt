What landed
- `apps/desktop/src/main/oracles.ts` no longer depends on the Bun runtime
  (`Bun.file().exists()` and `Bun.file().text()`). Replaced with
  `node:fs.existsSync` + structured error handling so the module runs cleanly
  in Electron's main process (Node).
- Added a `tryRunTmux` helper that returns `null` instead of throwing on:
  * tmux binary missing (ENOENT)
  * no tmux server on the socket
  * any other non-zero exit, with a debug log
- `listOracles` / `listTmuxSessions` now both return `[]` gracefully on
  no-tmux environments (covers the "no-tmux -> empty, not error"
  acceptance bullet).
- `deleteOracle` and `killTmuxSession` swallow the `can't find session`
  stderr so already-gone sessions are treated as success (idempotent
  cleanup after a forced kill).
- `renameOracle` and `createOracle` reject empty sanitized identities with
  clear error messages, mirroring the existing superapp contract.
- Replaced the proof-shell `oracle-panel.tsx` with a real surface that:
  * polls the live tmux roster every 5s via `useQuery`
  * lists oracles (master pinned + first), with a hidden set persisted to
    `localStorage` (`elf:oracle-panel:hidden`)
  * supports create / rename / delete with the master-delete safety (force
    confirm)
  * surfaces attached sessions and the all-tmux surface (other sessions)
  * has an "Attach" button that spawns a PTY pane against the oracle via
    `spawnPtyOracle` / `spawnPtyTmux`
  * shows explicit empty / error / loading states; no-tmux lands on the
    "No oracle sessions running" empty card, NOT a crash
