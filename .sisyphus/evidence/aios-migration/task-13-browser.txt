What landed
- Persisted last successful URL via Jotai `atomWithStorage` (`elf:browser-panel:last-url`),
  restored on panel mount so the inline browser keeps its destination across
  relaunches (covers the "state restores across relaunch" acceptance bullet).
- Persisted rolling history (8 most recent URLs) with de-dupe and a "Recent"
  chip row in the panel for one-click re-visit.
- Extracted URL building into `buildNavigableUrl(input): string | null` with
  helper tests:
  - Reject empty / whitespace / embedded-space input -> explicit error toast,
    not a silent "https://" prefix.
  - Pass through `http(s)://`, `file://`, `about:`, `chrome:`, `devtools:`.
  - Prefix bare hostnames (`example.com`) with `https://`.
- Wired `webview` `did-fail-load` and `did-finish-load` to surface real load
  failures (error code + description + URL) inside the panel instead of
  silently leaving the webview blank.
- Added a "Reset" action that clears the URL, history entry, and stored
  persisted URL back to `about:blank`.
- Added a "Go" submit button next to the input (form already had implicit
  submit on Enter; this is the explicit affordance).
- Browser-mode (no Electron) still falls back to the same "Inline browser
  needs Electron" empty state.

- Manual flow described for Chris:
  1. Open Browser surface; type `https://example.com`, hit Go -> load,
     persisted; "Recent" shows the URL.
  2. Type `not a url with spaces` -> explicit error toast in panel.
  3. Type `example.com` -> resolves to `https://example.com`, persists.
  4. Close + relaunch -> last URL restored; recent chips still present.
  5. Click a recent chip -> navigates; persists + de-dupes.
  6. Hit Reset -> back to `about:blank`, stored URL cleared.
  7. Toggle Browser feature flag off -> side panel fallback to another tab.
